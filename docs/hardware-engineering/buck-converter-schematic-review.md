# Buck 原理图评审要点

## 适用场景

适用于：

- 同步 buck
- 异步 buck
- 集成功率 MOSFET 的降压芯片
- 板级主电源或点电源的正式 schematic review

## 不适用场景

不适用于只检查 layout 细节的末期 SI/EMI 讨论。本文重点是原理图阶段能否提前避免大部分错误。

## 典型失效症状

Buck 设计没做对，现场最常见的表现是：

- 上电即过流或发热
- 输出电压能起来但纹波大、负载一拉就掉
- 满载稳定、轻载异常，或相反
- 开关节点振铃重，EMI 很差
- 反馈电压看起来正常，系统实际输出却飘

## 先看什么

不要先看电感值。先看三件事：

1. 功率环路是否闭合且清晰。
2. 反馈采样是否远离高 di/dt 回流。
3. 控制器辅助网络是否真的按 datasheet 约束实现。

## 必查顺序

### 1. 功率主回路

- 输入电容、上管、下管/续流路径、电感、输出电容组成的高 di/dt 环路是否短。
- `VIN`、`SW`、`PGND` 的连接是否符合典型应用。
- 是否存在大电流路径与敏感采样网络共用返回路径。

### 2. 反馈与补偿

- 反馈分压是否从真实输出点取样，而不是从噪声节点附近取样。
- 下分压是否落在信号地，而不是功率地回流末端。
- 补偿元件是否按器件推荐值和拓扑实现。

### 3. 辅助与保护

- Bootstrap 电容和二极管是否正确连接。
- EN、PG、SS、OC、ILIM 等脚是否有明确系统用途。
- 软启动、限流、打嗝模式、短路行为是否与系统预期一致。

### 4. 参数边界

- `Vin_min`、`Vin_max`、`Iout_max`、频率、占空比、温升是否在芯片工作区间内。
- 输出电容 ESR、纹波电流、容值衰减是否被纳入考虑。
- 电感饱和电流和 DCR 是否满足最坏负载。

## 硬规则

- `SW` 节点必须被当作噪声源，不允许在原理图上把它混入普通模拟/控制网络周围。
- 反馈采样不允许借道大电流地返回。
- 输入电容必须服务功率回路，而不是只服务“BOM 看起来有输入电容”。
- 任何偏离典型应用的外围改动，都必须能解释为什么不会破坏稳定性。
- 轻载、重载、启动、短路四种状态都要有预期行为。

## 常见失误

- 反馈下分压接到功率地。
- 输出电容换料后没重新检查 ESR 和稳定性。
- 只按典型应用复制，但目标输入范围和负载远超样例。
- 把 PG/EN 连上了，却没有和系统时序或故障处理闭环。
