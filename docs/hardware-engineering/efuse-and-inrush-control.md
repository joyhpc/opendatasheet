# eFuse 与浪涌电流控制

## 适用场景

适用于模块输入保护、板卡插拔、USB/工业口供电、服务器板卡和高可靠电源入口。

## 关键规则

- eFuse 的价值不只是过流保护，还包括限流模式、短路响应、故障报告和热保护。
- 必须区分恒流限流、折返限流、断路重试三种行为对系统的影响。
- 输出大电容充电是最常见的误判场景，限流和启动时间要一起设计。
- 输入源能力不足时，eFuse 设置不当会让系统反复打嗝而不是稳定启动。

## 评审清单

- 限流阈值是否高于正常启动峰值但低于上游可承受能力。
- 故障重试策略是否会与系统 MCU 或 PMIC 状态机冲突。
- 热保护触发后是否有明确恢复路径。
- 反向电流阻断和欠压锁定是否满足系统掉电要求。
- 故障指示脚是否被日志或控制逻辑真正使用。

## 常见失误

- 把 eFuse 当作“更聪明的保险丝”，却不审视其动态工作模式。
- 忽略大输出电容导致的充电时间，误以为器件启动异常。
- 保护阈值设置太紧，正常插拔时反而频繁误触发。
