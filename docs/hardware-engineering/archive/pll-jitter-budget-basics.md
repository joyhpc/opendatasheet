# PLL 抖动预算基础

## 适用场景

适用于 SerDes、以太网、ADC 采样时钟、视频接口和高性能 FPGA 时钟树。

## 关键规则

- 抖动预算必须端到端看：参考源、分配器件、PLL、供电噪声、布线和接收器容限。
- 不能把不同带宽、不同统计定义的 jitter 数字直接相加。
- 时钟清洗器件不是万能药，错误的环路带宽设置会放大系统问题。
- 对采样系统，近端相位噪声往往比宽带 RMS jitter 更关键。

## 评审清单

- 关键链路的接收端 jitter 容限是否已知。
- 参考时钟源噪声谱和 PLL 带宽选择是否匹配。
- 供电噪声是否通过 LDO、滤波或分区控制进入 PLL/VCO。
- 时钟扇出器件是否引入不必要的附加抖动或 skew。
- 文档中的 jitter 单位和积分带宽是否统一。

## 常见失误

- 用不同 datasheet 里的典型 jitter 数字直接横向比较。
- 只看总 RMS，不看相位噪声频段对系统的实际影响。
- 把时钟树放在脏电源上，希望靠后级 PLL 全部修复。
