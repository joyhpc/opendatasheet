# SPI 总线评审指南

## 适用场景

适用于 Flash、ADC、DAC、显示控制、传感器和板内高速短距串行链路。

## 关键规则

- SPI 没有统一总线仲裁，芯片选通、时钟极性相位和时序预算必须由系统定义清楚。
- 当时钟升高后，拓扑、stub、负载数量和驱动边沿都会变成主要问题。
- 多从设备共享总线时，未选通器件的 MISO 行为必须确认不会冲突。
- 长线 SPI 通常不是“稍微加大驱动”就能解决，必要时应降速或改接口。

## 评审清单

- CPOL/CPHA 是否与所有从设备匹配。
- CS 默认态、上拉下拉、上电瞬态是否避免误命令。
- 多从设备总线长度和 stub 是否可控。
- 关键路径是否需要串阻以减小过冲和振铃。
- 是否明确最大工作频率对应的板级时序预算。

## 常见失误

- 多个器件 MISO 默认驱动，未选通时仍争总线。
- 只按芯片支持的最高 MHz 宣称设计可行，未看板级拓扑。
- 将 SPI 拉出板间连接器却没有重新做 SI 评估。
