# 原理图分页组织方法

## 适用场景

适用于中大型板卡、多人协作、设计评审和后期维护。

## 关键规则

- 分页应服务功能分区、调试路径和 review 效率，而不是单纯按器件数量平均分配。
- 电源树、时钟树、处理器核心、接口块、模拟前端、连接器应有清晰页面边界。
- 同一功能链的关键控制和状态不要拆到完全无关页面。
- 页面结构必须让评审者能顺着信号流或电源流快速走通。

## 评审清单

- 页面标题是否能准确表达功能，而不是“Sheet 7”这种无意义命名。
- 电源、地、复位、时钟是否有统一入口页或明显交叉引用。
- 跨页网络是否通过层级端口或规范命名保持清晰。
- 高风险功能块是否能在有限页数内完整审视。
- 页面内器件和网络布局是否支持左到右、上到下的阅读逻辑。

## 常见失误

- 为了“页面均匀”把一个功能模块拆成多页碎片。
- 电源和控制网络到处跨页，评审时难以闭环。
- 页面只按原理图库默认排序，完全不服务调试和 review。
