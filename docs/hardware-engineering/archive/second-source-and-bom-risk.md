# 第二来源与 BOM 风险控制

## 适用场景

适用于量产项目、长周期供货项目、成本优化和器件停产预案。

## 关键规则

- 第二来源不是“封装看着一样就行”，而是电气、时序、启动、保护和软件兼容的整体问题。
- 风险最高的器件通常是 PMIC、时钟、FPGA、PHY、特殊接口芯片和精密模拟器件。
- 在设计冻结前就应标记单一来源高风险器件。
- 替代料策略要分为 `pin-to-pin`、`功能兼容`、`需改板/改软件` 三类。

## 评审清单

- BOM 中哪些器件没有第二来源且 lead time 长。
- 替代料是否共享关键参数、封装热性能和启动行为。
- 高风险器件是否预留了 footprint、阻容 option 或 strap option。
- 软件驱动、寄存器地图、时钟树和初始化序列是否绑死到单一器件。
- 采购和工程对风险等级的定义是否一致。

## 常见失误

- 只对比引脚，不对比电源时序和默认状态。
- 认为通用 LDO、TVS、MOSFET 随便替换没有系统影响。
- 高风险器件到缺货时才开始寻找备选。
