# 前后端交接清单

## 适用场景

适用于原理图到 layout、layout 到 SI/PI、layout 到 bring-up 的团队交接。

## 关键规则

- 交接不是把文件丢过去，而是把约束、风险和优先级一起交出去。
- 原理图阶段至少要输出关键网络分组、阻抗目标、长度规则、电源层级和高风险器件说明。
- layout 完成后必须把实际实现与原约束偏差重新反馈给系统和 bring-up 团队。
- 交接不清，问题就会在打样后变成“谁都以为别人知道”。

## 评审清单

- 是否有明确的网络类别和布局约束表。
- 高速接口、DDR、时钟、电源去耦的关键布局意图是否文字化。
- 是否列出不可更改的器件摆位和可调整区域。
- layout 结束后是否输出长度、层别、过孔和 stackup 实现结果。
- bring-up 团队是否拿到了最新原理图、BOM、layout 截图和约束说明。

## 常见失误

- 原理图只给 PDF，不给可执行约束表。
- layout 修改了关键器件位置，但没有回传影响分析。
- bring-up 用的是旧版 pinout 或旧版 BOM。
