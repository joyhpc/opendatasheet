# 测试点策略

## 适用场景

适用于 bring-up、量产测试、故障定位和场返分析的原理图与 PCB 预留。

## 关键规则

- 测试点不是“布板最后有空间就加”，而是调试和量产策略的一部分。
- 电源、复位、时钟、关键总线、模拟基准和故障指示应有分层可测性。
- 高速差分对通常不应直接挂长 stub 测试点，必要时应使用专用探测结构。
- 如果一个信号出问题会阻塞 bring-up，它就应该有可测路径。

## 评审清单

- 所有关键电源 rail 是否可测，且有地参考点。
- 复位、JTAG、UART、I2C、SPI 等 debug 通道是否可直接接入。
- 模拟关键节点是否预留高阻抗测量位置。
- 量产 ICT/FCT 与实验室调试测试点是否存在冲突。
- 测试点命名和原理图标注是否足够清晰。

## 常见失误

- 只给电源加测试点，不给状态和控制信号留观测手段。
- 高速线随意引出 stub 作为测试点，破坏信号完整性。
- 量产测试要求直到样板完成才被提出，导致返板。
