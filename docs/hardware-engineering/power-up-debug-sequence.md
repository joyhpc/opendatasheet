# 首次上电调试顺序

## 适用场景

适用于新板首电、返修板复测、未知故障板排查和量产前样机验收。

## 关键规则

- 上电调试必须按“静态检查 -> 分 rail 验证 -> 时钟复位 -> 基础接口 -> 功能链路”顺序推进。
- 首电目标不是立刻跑应用，而是确认不会继续损坏硬件。
- 每一步都应有明确通过条件和下一步进入条件。
- 有问题时先回到最近一个已知正确状态，不要并行乱改多个变量。

## 评审清单

- 上电前是否做短路、极性、阻值和视觉检查。
- 首次上电是否使用限流和分阶段供电。
- rail、电流、复位、时钟、Boot strap 是否按顺序验证。
- 关键通信口是否先验证最小闭环，如 UART 日志或 JTAG IDCODE。
- 每轮改动是否有记录，不让故障状态失去可追溯性。

## 常见失误

- 板子一到手就整机满功率上电。
- 时钟和复位都没确认就开始抓软件问题。
- 一次改多个硬件飞线和软件设置，结果无法定位根因。
