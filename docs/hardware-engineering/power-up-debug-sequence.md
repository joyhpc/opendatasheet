# 首次上电调试顺序

## 适用场景

适用于：

- 新板首电
- 返修板复测
- 量产前样机验收
- 未知故障板的硬件层排查

## 不适用场景

不适用于功能软件已经稳定、只做性能优化的阶段。本文针对的是“板子是否健康、能否安全往下走”的最早阶段。

## 典型错误做法

下面这些做法会显著拉高调试成本：

- 板子一到手就整机满功率上电
- 电源、时钟、复位都没看就开始追软件
- 多个飞线、多个寄存器设置、多个固件版本一起改
- 没有限流，没有记录，也没有分阶段通过条件

## 推荐顺序

### 1. 断电静态检查

- 视觉检查：器件方向、焊接、漏装、短路、连接器方向
- 阻值检查：主电源对地、关键 rail 对地、疑似短路点
- 极性检查：电解、电池、TVS、MOSFET、二极管方向

### 2. 受控上电

- 用可限流电源，不要直接整机大电流硬上
- 先低风险 rail，再高风险核心 rail
- 记录空载电流、启动电流、稳定电流

### 3. 电源树确认

- rail 是否按预期电压出现
- 是否有 rail 起不来、过冲、掉压、异常纹波
- 是否存在 PG、EN、fault 信号逻辑不对

### 4. 时钟与复位确认

- 主时钟是否存在且频率正确
- 复位是否被正确拉住并释放
- strap、boot mode、配置脚是否在正确状态

### 5. 最小通信闭环

- 先打通最小接口，例如 UART 日志、JTAG IDCODE、I2C 扫描、SPI 读 ID
- 不要一开始就追完整业务链路

### 6. 功能链路逐段放开

- 每次只增加一个变量：一条 rail、一个时钟域、一个外设、一个软件步骤
- 每通过一步，就记录波形、日志和判断结果

## 硬规则

- 首电目标是“确认不会继续损坏板子”，不是“尽快跑起来”。
- 每一步都必须有进入条件和退出条件。
- 有异常就回到最近一个已知正确状态，不做并行试错。
- 限流、电压、电流、温升、时钟、复位是首电六个必看项。
- 任何改动都必须可追溯：改了什么、为什么改、改后现象是什么。

## 常见失效症状与优先排查

- 上电立刻大电流：先查短路、极性、MOSFET 方向、PMIC 失控
- rail 正确但无启动迹象：先查时钟、复位、strap、配置模式
- JTAG 不通但 rail 正常：先查参考时钟、复位释放、JTAG 连接和电平
- 软件偶发能跑：先查电源时序、纹波、掉压、边界条件，而不是先怪固件
