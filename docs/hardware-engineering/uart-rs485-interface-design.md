# UART 与 RS-485 接口设计

## 适用场景

适用于调试串口、工业半双工/全双工通信、长线缆连接和现场总线节点。

## 关键规则

- UART 板内调试口重点看电平、参考地、默认态和保护；RS-485 重点看共模范围、终端和偏置。
- 半双工 RS-485 必须明确方向控制时序和总线空闲偏置。
- 长线缆环境下，隔离、电涌、ESD 和地电位差通常比波特率更关键。
- 终端电阻不是“看起来像差分对就加”，应结合拓扑和节点位置决定。

## 评审清单

- UART 电平是否与 MCU/USB 转串口器件兼容。
- RS-485 A/B 命名是否在全系统保持一致，避免现场接反。
- 总线首尾是否正确终端，中间节点是否不加终端。
- 空闲偏置电阻是否足以定义总线默认态。
- 是否需要隔离收发器和独立隔离电源。

## 常见失误

- RS-485 每个节点都加终端，结果驱动过载。
- 方向控制由软件硬拉，但没有考虑中断延迟和帧尾时序。
- 现场接口缺少共模和浪涌防护，只做了板内 ESD。
