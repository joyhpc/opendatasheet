# 复位与监控电路设计

## 适用场景

适用于 MCU、SoC、FPGA、PHY、PMIC 和多电源系统的上电管理。

## 关键规则

- 复位是时序问题，不是简单的“拉低一下”。
- 需要明确谁发起复位、谁接收复位、释放条件是什么、最坏上电时间是多少。
- RC 复位仅适合简单场景；多 rail 和严苛时序系统应使用 supervisor 或 PMIC 状态机。
- 监控门限、延迟和复位脉宽必须与被控器件 datasheet 一致。

## 评审清单

- 上电、掉电、欠压、按键和软件复位路径是否互不冲突。
- 开漏复位网是否有明确上拉和默认态。
- 不同电压域之间的 reset 是否安全跨域。
- 复位释放是否依赖稳定时钟、稳定电源或 PG 信号。
- supervisor 的阈值和迟滞是否适合系统容差。

## 常见失误

- 多个推挽复位源直接并联。
- supervisor 监控的不是最关键 rail，导致系统“看似复位正常”。
- 上电时间拉长后，RC 复位提前释放。
