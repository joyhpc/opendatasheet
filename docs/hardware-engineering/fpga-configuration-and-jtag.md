# FPGA 配置与 JTAG 设计

## 适用场景

适用于板载 Flash 配置、JTAG 下载、量产烧录和现场调试接口规划。

## 关键规则

- 配置模式、启动源、MSEL/BOOTMODE/strap 必须在原理图阶段定死并可验证。
- JTAG 口既是开发接口，也是故障恢复接口，不能只为“实验室可用”设计。
- 配置 Flash、复位、时钟和电源时序需要一起审视。
- 量产、调试、现场升级三种场景的访问路径要分清楚。

## 评审清单

- 配置管脚的上下拉是否满足上电采样窗口。
- JTAG TCK/TMS/TDI/TDO 是否有正确上拉下拉和连接器定义。
- 板载 Flash 容量、IO 模式、电压域是否匹配 FPGA 启动方式。
- 复位键、PROGRAM_B、nSTATUS、CONF_DONE 等状态路径是否可观测。
- JTAG 链中若有多器件，链顺序和旁路策略是否明确。

## 常见失误

- 只把 JTAG 接到测试点，实际无法稳定下载和边界扫描。
- 上电采样 strap 被其他逻辑或弱上拉干扰。
- 配置 Flash 与 FPGA IO 电平不匹配，开发阶段偶尔能起，量产不稳。
