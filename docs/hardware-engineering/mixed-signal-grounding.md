# 模拟与数字混合接地

## 适用场景

适用于：

- ADC / DAC 系统
- 精密参考与传感器前端
- 音频与 mixed-signal MCU 板
- 同时包含高速数字和精密模拟的控制板

## 不适用场景

不适用于拿“单点接地”当教条复制到所有板子。本文的重点是回流路径管理，不是形式化分地。

## 典型失效症状

混合地策略有问题，常见表现是：

- ADC 噪声底莫名升高
- 采样值受数字活动明显调制
- 某些模式正常，一开时钟或开关电源就异常
- 板子分了 AGND / DGND，但效果反而更差
- 实验室静态测量看着正常，系统动态运行就不稳定

## 先看什么

先不要讨论“到底分不分地”。先看：

1. 数字回流究竟走哪。
2. 模拟输入和参考周边是否有连续参考面。
3. 是否有人为切断了本该就近闭合的回流路径。

## 必查顺序

### 1. 回流路径

- 高速数字回流是否穿过模拟区。
- 时钟、开关节点、大电流路径是否压在精密模拟输入和参考附近。
- 是否存在跨分割布线却没有就近回流通路。

### 2. 芯片级要求

- ADC / DAC / mixed-signal IC 的 AGND / DGND 管脚是否按器件手册连接。
- 是否把 datasheet 中推荐的“芯片附近相连”误做成整板强分地。
- 参考源、输入缓冲、ADC/DAC 是否共享干净且连续的局部参考。

### 3. 地域关系

- 模拟地、数字地、保护地、机壳地、屏蔽地关系是否清晰。
- 是否因“图纸上看起来整齐”而把本不该硬连的域绑死。
- 是否因“怕噪声”把应该连续的参考地切成两半。

## 硬规则

- 混合接地的核心是回流控制，不是地平面形式主义。
- 若分地会切断关键返回路径，就不要分。
- AGND / DGND 的连接方式必须服从具体器件手册，而不是口口相传的经验。
- 高速数字和开关电源节点必须远离参考源、模拟输入和敏感运放区。
- 没看回流，只看铺铜外观，基本等于没做混合地 review。

## 常见失误

- 只因为看到 AGND / DGND 就强行整板分成两块地。
- 数字时钟线从参考源或模拟输入下方穿过。
- 让模拟输入回流绕远路才回到“单点地”。
- 保护地、机壳地、信号地关系说不清，EMI 和精度一起出问题。
