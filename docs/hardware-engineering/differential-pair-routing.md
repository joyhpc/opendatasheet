# 差分对走线原则

## 适用场景

适用于：

- USB 高速线
- Ethernet 差分线
- LVDS / SerDes / MIPI
- 高速差分时钟
- FPGA / PHY / connector 间的高速互连

## 不适用场景

不适用于低速、边沿不敏感、长度极短的普通 GPIO 成对布线。不要把“两个相关信号”都当差分对。

## 典型失效症状

差分对处理不当，系统上通常表现为：

- 链路能通但 BER 高、裕量低
- 高温或长线缆条件下才失败
- 眼图差、过冲大、EMI 重
- 某一方向或某一速率档位异常
- 插上连接器或加测试工装后性能明显恶化

## 先看什么

先看下面四项，而不是先纠结蛇形：

1. 目标阻抗是多少。
2. 返回路径是否连续。
3. 有没有无意义 stub 和多余换层。
4. 对内 skew 是否真的接近协议容限。

## 必查顺序

### 1. 约束来源

- 目标阻抗是否来自协议、器件手册或连接器/线缆规范。
- 是否区分了不同协议的 85 Ω / 90 Ω / 100 Ω 等要求。

### 2. 参考平面

- 差分对是否一直有连续返回参考。
- 是否跨过平面分割、大开槽、连接器断层或稀疏参考区域。

### 3. 拓扑与过孔

- 是否有长 stub、飞线式测试点、居中的 AC 耦合件。
- 是否一侧频繁换层、另一侧基本不换层。
- 过孔 stub 是否会进入目标频段问题区。

### 4. 对内匹配与耦合

- 对内 skew 是否在接收端预算内。
- pair gap 是否在工艺和阻抗设计窗口内。
- 是否为了补长度加了过多蛇形，反而引入更多损耗和串扰。

### 5. 周边器件

- AC 耦合、电阻网络、ESD、共模件是否对称放置。
- 连接器 pinout 是否服务返回路径和差分连续性，而不是只看机械方便。

## 硬规则

- 差分对的第一目标是通道完整性，不是“肉眼对称”。
- 参考平面连续性优先级高于局部长度强行完全一致。
- 不允许在高速差分线上随意挂长 stub 测试点。
- 不同协议的差分规则不能混用。
- 如果 layout 需要多次换层，必须同时考虑过孔 stub 和回流切换。

## 常见失误

- 为了长度匹配做过量蛇形。
- 只看 pair 内长度，不看整个通道里的连接器、过孔和回流断点。
- 将差分对走得“很整齐”，但穿过了最差的平面切割区。
