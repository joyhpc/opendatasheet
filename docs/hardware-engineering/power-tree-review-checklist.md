# 电源树评审清单

## 适用场景

适用于板级原理图评审、电源树规划复核、模块 bring-up 前的第一轮结构检查。

## 关键规则

- 先确认输入源、保护链路、分配节点、末端负载四层关系是否清晰。
- 每一路电源都应明确 `输入范围`、`目标电压`、`最大负载电流`、`启动时序`、`容差预算`。
- 不要把“能从同一母线分出来”误当成“可以共用同一稳压器”。
- 数字核电源、模拟参考电源、噪声敏感时钟电源应单独审视，不要只看额定电流。

## 评审清单

- 输入电源是否有反接、浪涌、ESD、热插拔或限流保护。
- 每个 regulator 的输入和输出范围是否覆盖最差工况。
- 上游 rail 掉压后，下游 rail 是否可能被外部 IO 或反馈网络反向灌电。
- 是否有必须先起或后起的 rail，被 enable、PG 或 sequencer 正确约束。
- 大电流 rail 的回流路径、铜皮宽度、测试点和电流采样位置是否明确。

## 常见失误

- 只看稳压器额定输出电流，不看热损耗和最差输入电压。
- 忽略 FPGA、DDR、SerDes 的电源噪声分组，导致 rail 规划先天混乱。
- 把模拟和数字负载挂在同一“方便”的 LDO 上，后期再靠 layout 勉强补救。
