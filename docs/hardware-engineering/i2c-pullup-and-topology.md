# I2C 上拉与拓扑设计

## 适用场景

适用于板内 I2C、跨板连接、热插拔模块、传感器总线和低速配置总线。

## 关键规则

- I2C 是 RC 边沿系统，先算总线电容和上升时间，再选上拉阻值。
- 同一总线上的所有器件必须共享兼容电压域，跨域要用专用 level shifter。
- 长线缆、连接器和多板拓扑会显著增加容性负载，不能沿用板内默认上拉。
- “能通信”不代表设计正确，边沿裕量和噪声裕量必须留足。

## 评审清单

- 上拉阻值是否同时满足上升时间和下拉电流能力。
- 总线是否被多个模块重复上拉，导致等效阻值过小。
- 开漏中断、复位、GPIO 扩展口是否被误接成推挽。
- 跨连接器 I2C 是否增加了 ESD、防呆和地参考策略。
- 地址冲突和热插拔时序是否被系统软件覆盖。

## 常见失误

- 认为 4.7 kΩ 是万能默认值。
- 忽略 level shifter 双向行为和时序限制。
- 板间 I2C 没有地参考或回流路径规划。
